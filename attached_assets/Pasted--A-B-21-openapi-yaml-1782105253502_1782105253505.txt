# 작업: (A) 학습 효과 = 정확도 추이 곡선 전면화  (B) 진영 배지 실제 우세 기반 + 21대 데이터 정합성

## 전제
- 스택/규약 동일. openapi.yaml→codegen, jsonReady(), 카피 한국어. 기존 기능 후퇴 금지.

## A) 합성인구 학습 — 토글 유지 + 정확도 추이 곡선 대표화
- 입력 보정(applyToPopulation) 토글은 그대로 둔다(정보성). 제거하지 말 것.
- "검증·정확도" 페이지(또는 합성인구 학습 탭) 상단에 "정확도 추이" 라인차트를 학습 효과의 대표 지표로 전면 배치한다:
  - x축: 검증 이벤트 기준일(electionDate/이벤트 기준일) 오름차순.
  - y축: 원시 오차(%p, 학습 전). 추세선이 내려갈수록 학습이 효과 있다는 의미.
  - 보조 표기: 평균 원시 오차(대표 숫자, 크게), 이벤트 수.
  - 캡션: "검증 이벤트가 쌓일수록 학습 전 예측 오차가 줄어드는 추세를 보여줍니다. (입력 보정 ON 기준)"
- 대시보드의 정확도 추이와 동일 데이터·동일 계산을 재사용(중복 로직 금지, 공유 훅/함수).

## B) 진영 배지 (b) + 21대 데이터 정합성
### B-1) 모든 선거 표시를 제21대 대선으로 통일 (화면=시드 일치)
- 라이브 DB의 제20대(2022-03-09) 선거 행을 제거하고, 아래 제21대(2025-06-03) 17개 행으로 재시드한다.
- 화면·코드에서 하드코딩된 "제20대"/"2022-03-09" 문자열을 모두 "제21대 대통령선거"/"2025-06-03"으로 교체. 선거 import 섹션의 기본 라벨도 21대로.
- metric 라벨은 "보수 후보(김문수) 득표율" 유지.

### B-2) elections 스키마에 실제 우세 진영 필드 추가
- 컬럼 추가: actualWinner text ('conservative' | 'progressive')  // 그 지역 실제 1위 진영. 기존 leaning('지표가 측정하는 진영', 전부 conservative)은 그대로 두고 별도 컬럼.
- `pnpm --filter @workspace/db run push`.

### B-4) 진영 배지 = actualWinner 기반 (calibration page)
- 기존 `r.leaning === "conservative" ? "보수" : "진보"` 로직을 폐기.
- 새 배지: actualWinner === "conservative" ? "보수 우세"(Badge destructive/red) : "진보 우세"(Badge default/blue).
- **득표율(actual_value) >50 임계로 판정하지 말 것** — 강원·울산이 50% 미만 김문수 우세라 오표시됨.
- 컬럼 헤더는 "우세 진영"으로. 카드 설명에 "값은 보수 후보(김문수) 득표율이며, 배지는 해당 시·도 실제 1위 진영입니다." 1회 표기.

## 완료 기준
- 정확도 추이 곡선이 학습 탭/대시보드 상단에 대표 지표로 보인다(입력 보정 토글은 유지).
- 모든 선거 표기가 제21대(2025-06-03)로 일치하고 20대 흔적이 없다.
- 진영 배지가 실제 우세 기준으로 표시된다(강원·울산=보수 우세, 광주·전남=진보 우세, 대구·경북=보수 우세).
- `pnpm run typecheck` 통과, 기존 기능 무손실.
